Hi everyone,

We’re going to be working together to build a simple academic website. We’re specifically going to build a website that looks like this one: https://www.ktensor.com. To expedite the process, please come with the following:

Stuff related to you and your work

  • Your CV in pdf format.
  • A headshot (preferably a professional one, but a reasonably good photo will do for now).
  • Your contact information, specifically:
    • Your email address
    • Your twitter handle (if desired)
    • Your department’s mailing address
  • For each publication you would like to advertise, the following information as applicable:
    • The title of the work
    • A list of co-authors
    • Any citation information
    • A very brief (two line) description of what you studied
    • Either: a pdf copy of the preprint _or_ a link to an open access copy of the work (e.g., on arxiv.org, soxarxiv.org, or an open access journal).
      • SSRN has a paywall. This doesn’t count.
      • NBER has a paywall. This doesn’t count.
      • Links to Elsevier websites don’t count.
      • A link to Google Drive, Dropbox, or any other free commercial file server also doesn’t count. These are often blocked by institutional employers.
  • Any links you want associated with this publication, including media reports, public data sets, or congressional testimony.

A Domain Name or Domain Name Idea

We will also be setting up a domain name for you. Depending on how common your name is, this can be hard to get one you want. Before the webinar, please go to WHOIS and check if your desired domain name is available.

If you already have purchased a domain name, note that I can’t provide click-by-click instructions for each registrar. However, most domain registrars are pretty similar. We will be using the Google domain name registrar.

A Personal Google Account

We are going to be building everything on Google Sites and using GitHub as a paper repository. Do not set this up on your institutional account. Unless you are a tenured professor not planning on moving anytime soon, you’ll want your website to follow you and not stay at your old institution. As such, please make sure you have a gmail account.

To set one up, go to gmail.com and click "Create an Account."

A GitHub Account

We're going to be using GitHub as a file server for all your papers. Please make sure you have a GitHub account that you can access. If you do not, please go to github.com and click the "Sign Up" button.

Ability to Video Conference with Zoom

We'll be using Zoom for the video conference. Please make sure that you have the ability to join a Zoom meeting before the day of the webinar.

Some Disclaimers

This tutorial will be recorded. We intend to put it up on YouTube or Vimeo so others can use it to put their website together in the future. If you are uncomfortable with this, please DM @khayeswilson on Twitter and we’ll try to make something work.

Also, this is going to be a group session where we go step by step. That means it will be a little slow and also you’re going to end up with a very generic website. Any further styling is up to your own creativity.
